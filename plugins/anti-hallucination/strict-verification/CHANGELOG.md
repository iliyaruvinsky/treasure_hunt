# Changelog

All notable changes to the Strict Verification Rules plugin will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.0.0/),

## [1.0.0] - 2025-10-14

### Added
- Initial release of Strict Verification Rules plugin
- Rule 1: Verify Before Claiming
- Rule 2: No Assumptions as Facts
- Rule 3: Mandatory Verification Workflow
- Rule 4: Honest Reporting
- Rule 5: Cost Consciousness
- Rule 6: No Confidence Without Verification
- Rule 7: Anti-Hallucination Mandate
- Rule 8: No "Yesman" Behavior
- Rule 9: Truth as Highest Value
- Rule 10: File Reading Status Protocol
- Enforcement mechanisms and violation consequences
- Mandatory process for single and multiple file edits
- Summary reporting guidelines

### Compatibility
- Claude Code: ✅ Fully supported
- Cursor: ✅ Fully supported
- Windsurf: ✅ Fully supported
- Cline: ✅ Supported
- Aider: ✅ Supported

### Notes
- This is the foundational version based on battle-tested rules
- No exceptions to these rules under any circumstances
- Designed for production use in enterprise environments
